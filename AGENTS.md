# TubeLens AI Engineering Guide

This repository is designed for AI-assisted development.

Before making implementation decisions, understand the project through the documents inside the `context/` directory.

Read these documents in order:

1. context/PRODUCT.md
2. context/CONSTITUTION.md
3. context/PHILOSOPHY.md
4. context/AI_BEHAVIOR.md
5. context/ROADMAP.md

Do not assume generic implementation patterns.

TubeLens follows a product-first development methodology.

Every implementation should support the project's guiding word:

Understanding.

Whenever uncertain:

- Prefer clarity over complexity.
- Prefer meaningful interactions over visual effects.
- Prefer long-term maintainability over quick solutions.

Do not introduce features that conflict with the project's philosophy.

Before implementing a feature, ensure you understand why it exists.
